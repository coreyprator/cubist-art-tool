Cubist Production Runner — Help

This guide explains the Production Runner (prod_ui.py): input selection, parameters, built-in geometries, discovered plugins, and actions. It also shows how to open the gallery and where outputs are written.

This page is generated/managed by the app. You can safely edit it to add screenshots or updated examples.

Quick Start

  1. Choose an Image (JPG/PNG) at the top left.
  2. Set Points, Seed, and optional Cascade Stages.
  3. Pick one or more Geometries and any Plugins you want to run.
  4. Check Export SVG if you want SVG files.
  5. Click Run Selected.
  6. Use Open Output Folder or the new Open Gallery button.

Fields

FieldWhat it does
ImageSource input used by the CLI for sampling, seeding, and color fills (when applicable).
PointsNumber of seed points or grid density. Higher values → more detail, slower render.
SeedRandom seed for reproducible results.
Cascade StagesNumber of refinement passes. E.g. 3 performs three sequential stages.
Export SVGWrites .svg files alongside any raster outputs.
Enable plugin execAllow discovered geometry plugins (e.g. poisson_disk).

Gallery

Use Open Gallery to generate gallery.html inside the selected output root and view all SVGs/PNGs side‑by‑side. This calls scripts/make_gallery.py <output_root>.

Tips

  • Re‑run with the same Seed for exact reproducibility.
  • Use --param seed_mode=poisson in the CLI for evenly‑spaced seeds.
  • For cascade styles driven by the image, prefer a moderate point count and 2–3 stages.
